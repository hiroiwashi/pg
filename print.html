ChatGPT を用いて FastAPI の勉強をした話

勉強スケジュールの提示

FastAPI

質問 1

fastapi の習得を 1 週間、react の習得を 1 週間で行い、1 週間でバックエンド fastapi・フロントエンド react でログイン機能付きの web アプリを作成したいです。 勉強の予定表を作成してください。

回答 1

以下は、1 週間の学習計画の一例です。これは、初心者の場合にはかなり濃密なスケジュールになっています。予定表は、学習の進捗状況や個人の学習スタイルに合わせて調整する必要があります。

Week 1

Day 1: FastAPI

  • FastAPI の概要の学習

Day 3: FastAPI

  • FastAPI での認証と認可の学習
  • FastAPI でのトークン認証の実装
  • FastAPI での JWT トークン認証の実装

Day 4: React

  • React の基本的な概念の学習
  • React のコンポーネントの作成とレンダリングの方法の学習

  • React でのサンプルアプリケーションの作成

Day 6: FastAPI + React

  • FastAPI と React の API とフロントエンドの連携の学習
  • FastAPI と React の統合アプリケーションの作成

Day 7: FastAPI + React

  • 統合アプリケーションの完成
  • ログイン機能の実装とテスト
  • ユーザーのデータの CRUD 操作の実装とテスト

この予定表には、以下のような注意点があります。

1 日に FastAPI または React について 6 時間以上を費やすのは、集中力や記憶の維持に悪影響を及ぼす可能性があります。必要に応じて、学習時間を短縮して休息を取ることが重要です。 統合アプリケーションの作成には、API とフロントエンドの両方を組み合わせるための技術的なスキルが必要です。一度にすべてを理解することは難しいかもしれませんが、一歩

質問 2

すみません。FastAPI で１週間、React で１週間、FastAPI + React で１週間です。 まず FastAPI から教えてください。

回答 2

了解です。FastAPI の学習計画の例を以下に示します。

Week 1: FastAPI

Day 1

  • RESTful API の基本的な概念の学習
  • FastAPI の概要と特徴の学習
  • FastAPI のインストールとセットアップ
  • FastAPI の基本的なルーティングと HTTP メソッドの処理

Day 2

  • FastAPI のモデルの作成とデータベース接続の学習
  • FastAPI での CRUD 操作の実装
  • FastAPI の例外処理とエラーハンドリング

Day 3

  • FastAPI での依存性注入の概念の学習
  • FastAPI でのセキュリティ対策の実装
  • FastAPI での OAuth2 認証とトークン認証の実装

Day 4

  • FastAPI のテストの概念とツールの学習
  • FastAPI のテストの実装
  • FastAPI の API ドキュメントの自動生成

Day 5

  • FastAPI の高度な機能の学習、例えば GraphQL、WebSocket、バックグラウンドタスク、Redis、Celery など
  • FastAPI のベストプラクティスの学習

  • 実際に FastAPI を使って、小規模な API サービスの開発と実装

  • 開発した API サービスのデプロイと公開

質問 3

Day6 で作るべき API サービスの要件を定義してください

回答 3

FastAPI の学習計画において、Day6 で作るべき API サービスの要件を以下に定義します。

Day 6: API サービスの開発

要件定義

  • API サービスは、ユーザー認証・認可が必要な機能を提供する
  • ユーザーは、サインアップ、サインイン、ログアウトができる
  • ユーザーは、アカウント情報の閲覧・更新ができる
  • ユーザーは、投稿の作成・閲覧・更新・削除ができる
  • 投稿には、タイトル、本文、投稿者、投稿日時が含まれる
  • 投稿には、複数の画像が添付できる
  • 投稿には、いいねがつけられる
  • ユーザーは、他のユーザーの投稿にいいねがつけられる
  • ユーザーは、自分がつけたいいねの一覧を閲覧できる
  • ユーザーは、他のユーザーのプロフィールを閲覧できる

上記の要件を満たす API サービスを開発することで、FastAPI の基本的な機能や機能拡張を学ぶことができます。また、フロントエンドの React との組み合わせで、完全な Web アプリケーションの開発に挑戦することができます。

mdbookプログラミング

自作のpreprocessorを使えるようにした

行ったこと

  • frontmatterを読みこむプリプロセッサーであるmdbook-frontmatterを改良した
  • githubactionsでページを作成する際にそれを使えるようにした

ディレクトリ構成とDockerfile

元のディレクトリは以下のものから始めた。

.
├── .github
│   ├── actions
│   │   └── build
│   │       ├── Dockerfile
│   └── workflows
│       └── main.yml
├── .gitignore

自分用にプロプリプロセッサを作ったため、それをgithubactionsで使用することにした。そのためにはどこかに置いてcargo installすれば良いことはわかっていたので、とりあえずルートにおいてDockerfileでのinstallを試みた。

.
├── .github
│   ├── actions
│   │   └── build
│   │       ├── Dockerfile
│   └── workflows
│       └── main.yml
├── .gitignore
├── mdbook-hiromatter #自作のプリプロセッサ

Dockerfileを使ってbuildを行うが、どうやってもcargo installができない。ディレクトリが参照できず、エラーを吐いて終わってしまう。

dockerのbuildを行う際は/.github/actions/buildの中を参照しているのだから、その中に入れることにした。

├── .github
│   ├── actions
│   │   └── build
│   │       ├── Dockerfile
│   │       ├── mdbook-hiromatter
│   └── workflows
│       └── main.yml
├── .gitignore

これでうまくいった。buildディレクトリの外側はbuild時に読み込めないので、その中に入れるしかないような気がするんだけどどうなのか。

本当はgithubにあげてしまってcargo install --gitしたかったんだけど、privateでしか作ってないから無理でした

部活が終わった今思うこと

先日４年生を送る会なるものが行われ、僕の４年間の部活生活は完全に終わった。前に書いたときはシーズン中でしっかり振り返りきれてないような気もするから、一応今の自分の考えを少し残しておこう。

競技自体も好きだし周りの人も当たり前に大学でラグビーを続けてたから、大学でラグビーで関わらないという選択肢はなかった。ただ、併願校的にも第一志望受からなかったら選手を続けるのは無理そうと思ってたから（実際にスタッフ希望のメールも送っていた。今思えば続ける勇気がなかっただけなんだけど。）、受かった時にこれでラグビーできるなと嬉しかったのは今でも覚えてる。合格発表の日にはラグビー部の人を見つけて話しかけに行ったし、入学式の前に入部届を出して入部した。ラグビーをすることが当たり前だったし入部することに何の悩みもなかったから、今振り返っても自分がした選択に対して何も思うことはない。今１年生になっても絶対ラグビー部を選ぶし、というか選ばざるを得ないというか。

この４年間、ラグビー出来るってだけで僕は満足だった。きつかったり辛いことは山ほどあったけど、少しやる気がなくなってしまったことはあったけど、一度もラグビーを辞めたいとは思うことはなかった。部活を続ける理由については４年間でよく議論されていて、勝つためとか人が好きだからとかあげようと思えば幾つでもあげられるけど、結局はラグビーが好きっていうのが根本にあったし一番大きな理由だった。やればやるほどラグビーの奥深さに触れることができたし、知ろうとすればするほど新たな顔を見ることができた。10 年以上やってきたけど新たな発見ばかりで飽きることがなかった。ラグビーは最高なスポーツだと確信している。

やっぱり僕はラグビーが好きだし、僕にはラグビーしかないんだな。後輩とかコーチで教えた子供たちとか、他の人たちがラグビーの魅力を知ってラグビーを好きになっていく姿を見ると、本当に嬉しいし幸せに思う。これから先ラグビーに出会う人たちに何を一番伝えたいかって考えると、もちろん部活とかで真剣にやるんだったら最後まで続けてほしいっていうのはあるんだけど、とにかくみんなにはラグビーを好きになって欲しい。好きになるまでは時間がかかるかもしれない。あんなにも辛くてきついスポーツはないのだから。だけどいつか好きになってくれたら、少しでも楽しいと思ってくれたら、ラグビーの先輩としてこれ以上幸せなことはない。

松任谷由実は歌う。
